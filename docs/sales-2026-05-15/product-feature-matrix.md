## 3. 데모 화면 순서 제안 — OS ↔ BO 왕복 스토리

> **설계 의도**
> OS(실무자 뷰)와 BO(관리자/대표 뷰)를 **한 거래 사이클 내에서 교차 시연**하여,
> 두 앱이 분리된 서비스가 아니라 **하나의 업무 흐름을 역할별로 분할한 유기적 시스템**임을 보여준다.
>
> 스토리 주인공:
> - 👤 **실무자(상담원)** — OS 앱 사용
> - 🧑‍💼 **관리자/대표** — BO 앱 사용

### 3-1. 데모 사전 준비

- **2개 브라우저 창 분할** (또는 듀얼 모니터): 왼쪽=OS(3000), 오른쪽=BO(3001)
- 각 창에 **다른 계정으로 미리 로그인** — OS는 상담원 권한, BO는 ADMIN 권한
- 시드 데이터: 골프장 1~2건, 고객 2~3건, 상담일지 1건(이전 이력용)

### 3-2. 시연 순서 (총 8막)

| 막 | 앱 | 화면 | 스토리 | 연결 포인트 |
|----|----|------|--------|-----------|
| **1막. 도입** | 🧑‍💼 BO | `/` 대시보드 | 관리자가 오늘의 KPI·공지·알림을 확인하며 하루 시작 | KPI 숫자 → 이 숫자가 어떻게 쌓이는지 이후 보여줄 예고 |
| **2막. 자산 DB** | 👤 OS | `/clubs` → `/clubs/[code]` | 상담원이 고객 문의 응대를 위해 골프장/회원권 정보 조회 | "같은 DB를 관리자는 어떻게 편집하는가?" 로 전환 |
| **3막. 자산 편집** | 🧑‍💼 BO | `/clubs/[code]` (5개 탭) | 관리자가 시나리오·서류·회원권 정보 인라인 편집 → 저장 | **OS를 새로고침하면 변경사항 즉시 반영** 시연 (실시간 연계 증명) |
| **4막. 고객 & 상담** | 👤 OS | `/customers` → `/trades` | 상담원이 신규 고객 등록 → 상담일지 작성 → **"승인 요청"** 제출 | 화면에 PENDING 상태 뜨는 순간 BO 창으로 전환 |
| **5막. 승인 워크플로우** ⭐ | 🧑‍💼 BO | `/trade-memos` | 관리자 화면에 방금 올라온 건이 **즉시 표시** → 검토 → 승인/반려 → 관리자가 거래 완료로 전환 | **양쪽 화면이 실시간으로 연결됨을 눈앞에서 증명** — 가장 강한 임팩트 구간 |
| **6막. 거래 확정** | 👤 OS → 🧑‍💼 BO | OS `/membership-trades` → BO `/trade-records` | 상담원이 승인된 건의 거래정보 입력(수수료·세금 자동계산) → 관리자가 최종검토 완료 | "입력은 현장, 검토는 본사" 역할 분리 어필 |
| **7막. 경영 가시성** | 🧑‍💼 BO | `/kpi` → `/` 대시보드 | 방금 완료된 거래가 KPI 차트·대시보드 숫자에 **즉시 반영** 확인 | 1막의 숫자가 어떻게 만들어지는지 **루프 종결** |
| **8막. 권한/조직** | 🧑‍💼 BO | `/users` → `/my-organization` | ADMIN/EDITOR/VIEWER 역할별 UI 차이 시연, 조직 단위 배포 메시지 | "귀사도 동일하게 세팅 가능" 으로 세일즈 클로징 |

### 3-3. 핵심 연결 포인트 (놓치면 안 되는 순간)

| 순간 | 보여줄 것 | 왜 중요한가 |
|------|-----------|-------------|
| 3막 → 2막 새로고침 | OS에서 수정된 골프장 정보가 바로 보임 | **같은 DB** 임을 시각적으로 증명 |
| 4막 → 5막 전환 | OS에서 "승인 요청" 누르는 순간 BO 목록에 **신규 항목 표시** | 실시간 데이터 흐름 증명 |
| 5막 승인 → 4막 복귀 | OS의 PENDING → APPROVED 상태 변화 | 워크플로우 역방향 동기화 증명 |
| 6막 OS 입력 → BO 검토 | 수수료 계산식이 양쪽에서 동일 | **비즈니스 로직 공유**의 가치 |
| 7막 KPI 반영 | 1막의 숫자가 거래 1건으로 변동 | 모든 활동이 경영 지표로 연결됨 |

### 3-5. 시연 팁

- **화면 전환 시 마우스 커서를 크게** 해두어 청중이 "지금 어느 앱인가" 혼동하지 않게 함
- 각 막 시작 시 **1줄 내레이션**: "이제 상담원 화면입니다", "이제 관리자 화면입니다"
- 5막에서 승인하는 순간 **의도적으로 잠깐 멈추고** 청중이 양쪽 화면 변화를 눈으로 확인하게 시간 확보
- 시세/마케팅 랜딩은 **이 8막 흐름에 넣지 않음** — 질의응답이나 별도 섹션으로 분리 권장

## 5. 확인 필요 항목 (기획/PM 입력 요청)

- "시세 화면"이 독립 페이지로 기대되는지, 현재 골프장 상세 내 통합으로 충분한지
- "대표 화면"이 현재 KPI/대시보드와 동일 범위인지, 별도 임원 뷰가 필요한지
- 데모 화면 순서가 세일즈 스토리와 맞는지
- OS `/claims`(건의사항), BO `/common-documents`·`/notifications` 데모 포함 여부
- 외부 공개용 마케팅 랜딩(Framer/Webflow 등) 별도 구축 여부
